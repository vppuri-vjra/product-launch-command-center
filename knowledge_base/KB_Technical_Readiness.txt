## Architecture Overview

### System Components
- **Frontend:** React Native mobile app (iOS + Android), Next.js web
- **Checkout Service:** Node.js microservice, handles cart operations and payment orchestration
- **Payments Gateway:** Stripe Payment Element (primary), PayPal (fallback)
- **Loyalty Service:** Python microservice, manages points, tiers, and redemption
- **Notification Service:** AWS SES for email, Firebase for push notifications
- **Database:** PostgreSQL (primary), Redis (caching, session management)
- **CDN:** CloudFront for static assets
- **Monitoring:** Datadog (metrics + logs), PagerDuty (alerting), Sentry (error tracking)

### API Performance (Current Benchmarks)
- Checkout API: p50 = 120ms, p99 = 340ms (target: p99 < 500ms) ✅
- Loyalty Points API: p50 = 45ms, p99 = 180ms ✅
- Payment Processing: p50 = 800ms, p99 = 2.1s (Stripe-dependent, within their SLA)
- Address Auto-complete: p50 = 90ms, p99 = 250ms ✅

## Deployment Plan

### Staged Rollout Schedule
- April 15 (Day 1): 5% of users — canary group, heavy monitoring
- April 18 (Day 4): 25% of users — if no critical issues from canary
- April 22 (Day 8): 50% of users — mid-rollout check
- April 25 (Day 11): 100% of users — full rollout

### Rollback Criteria (Automatic revert if ANY of these trigger)
- Payment failure rate exceeds 3% for 15 consecutive minutes
- Checkout API p99 exceeds 1 second for 10 consecutive minutes
- More than 50 Sentry errors of the same type within 5 minutes
- App crash rate exceeds 0.5%

### Rollback Process
- Feature flags disable all v3.0 features within 2 minutes
- Users seamlessly revert to v2.9 checkout experience
- Loyalty program features hidden (no data loss — points preserved in database)
- Total rollback time: under 15 minutes
- Rollback has been tested 3 times in staging environment
